Java 的编码问题

这个问题，很明显，这是一个文字编码的问题，在.java文件中所有的中文都无法正常显示，这是由于当时读取时使用的工具的编码与项目的保存时的编码不一致所致，当把编码方式调整之后，显示正常，

这个文档的编码方式是UTF-8，但是在不同平台上打开时，解码方式却有可能不同，
刚开始很头疼，因为不知道怎么在对方不做出手动调整的前提下做出修改，使对方可以正常读取文件，
于是上网查了很多资料，于是接触到一个属性，“file.encoding”,于是，我做了一个测试，

所以，基本可以确定file.encoding内存储的值就是开发工具设置的编码方式，
所以，把这个值改变就可控制开发工具在打开文件时使用的编码方式了吧？？？？？？？？？？？？

把encoding从UTF8改变为GBK，但是文字没有变化，但是charset的变化与eclipse编码设置改变一致，所以，看来可以从charset入手？？？？？？？，

结语

对于项目的开发者来说，未发现使用代码可以改变项目的编码方式，所以如果想要改变项目的编码需要在开发工具内部设置，对于多人跨平台开发项目来说，最好都使用同样的编码方式，否则将会出现项目无法对接的现象。
